Georg - Setup Configuration

Before diving into daily operations, you need to configure Georg to match your business needs. This page allows you to set up the essential components that will structure your complaint and claims management process.

Note: Setup should be completed before you begin using the system. However, you can return to these settings at any time to make adjustments as your business requirements evolve.

First-Time Configuration Steps

1 Setting Up Number Series

The number series determines how your complaints and claims will be numbered in the system:

  • Navigate to "Setup default values" in the main menu
  • Click "Add Default Number Series" to use the pre-configured numbering pattern (starting with ISSUE0001)
  • Alternatively, click into the "Complaints Nos." field to define your own custom numbering pattern

A well-designed number series can help with tracking and sorting issues. Consider your organization's existing numbering conventions when making this choice.

2 Importing Master Data

Georg comes with pre-defined master data to help you get started quickly:

  • Click "Add Default Master Data" to instantly populate your system with 42 generic default values

3 Customizing Master Data

After importing the default values, you can customize them to fit your specific business needs:

  1. Return to the main menu after completing the initial setup
  2. Click on any of the master data categories (Business Units, Reasons, Origins, etc.)
  3. Add, modify, or remove entries as needed to reflect your organization's terminology and structure

Pro Tip: When designing your master data structure, consider your reporting needs first. What information will management need to make decisions about process improvements?

Important: These defaults are designed as a starting point. You should review and customize them to match your specific business terminology and processes. Well-structured master data will make reporting and analysis much more effective.

Moving Master Data Between Systems

If you need to transfer your master data configuration between different Business Central environments:

  • Use the Export function to save your master data to a file
  • Use the Import function to load previously exported master data into a different system

This is particularly useful when setting up test environments or when deploying to multiple companies within your organization.
