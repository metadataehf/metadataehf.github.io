Georg - Master Data Management

Back to Main Help Page

Master data in Georg provides the structure and classification for your complaints and claims. Well-designed master data is essential for effective reporting and analysis, enabling you to identify patterns and make informed business decisions.

Types of Master Data

Business Units

Departments or functional areas in your organization responsible for handling specific types of complaints.

Examples:
  • Sales
  • Accounting
  • Warehouse
  • Customer Service
  • Technical Support

Assigns responsibility and enables department-specific reporting

Reasons

The specific issues that prompted the customer complaint.

Examples:
  • Invoice Error
  • Late Delivery
  • Product Defect
  • Incorrect Order
  • Missing Items

Identifies common problem areas and tracks issue frequency

Origins

How or where the complaint was initially received.

Examples:
  • Customer
  • Email
  • Phone Call
  • Website
  • Social Media

Helps understand how customers prefer to communicate issues

Categories

Broader groupings for strategic analysis of complaints.

Examples:
  • Billing
  • Shipping
  • Product Quality
  • Service
  • Communication

Enables high-level analysis and reporting of issue types

Types

Classification of complaint severity or priority.

Examples:
  • Critical
  • Regular

Helps prioritize which issues need immediate attention

Managing Master Data

On each master data page, you can perform the following actions:

Add new entries
Edit existing entries
Delete entries
Import data
Export data

Note: You cannot delete master data values that are currently in use by existing complaints or claims. This ensures data integrity and prevents orphaned records.

Best Practices for Master Data Management

  1. Be Consistent - Use clear, consistent naming conventions across all master data types. For example, use all uppercase for codes and proper case for descriptions.
  2. Limit Proliferation - Avoid creating too many similar categories or reasons. Having too many options can complicate reporting and make it difficult to identify patterns.
  3. Review Regularly - Periodically assess if your master data still reflects your business needs. As your business evolves, you may need to add, modify, or deactivate certain values.
  4. Document Meanings - Ensure all team members understand what each value represents. Consider creating a reference guide that explains when to use each option.
  5. Think About Reporting - Design your master data structure with reporting needs in mind. The way you categorize complaints will directly impact your ability to analyze trends effectively.

Pro Tip: When designing your master data structure, start by identifying what information management will need to make decisions about process improvements. This ensures your classification system captures the most valuable insights.

Moving Master Data Between Systems

Georg allows you to export and import master data, which is particularly useful when:

  • Setting up a new Business Central environment
  • Transferring configurations between test and production environments
  • Standardizing master data across multiple companies in your organization

To transfer master data:

  1. In the source system, use the Export function to save your master data to a file
  2. Transfer the file to the target system
  3. In the target system, use the Import function to load the master data
